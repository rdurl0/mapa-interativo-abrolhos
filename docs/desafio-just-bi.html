sales_weekly_by_customer <- "data/Dataset_teste_Just_BI.csv" %>%
  here() %>%
  read_delim(
    col_types = cols(
      customer_id = col_character(),
      customer_acquisition_channel = col_character(),
      year = col_integer(),
      week = col_character(),
      net_revenue = col_double(),
      gross_revenue = col_double(),
      boxes = col_integer()
    )
  ) %>%
  mutate(week_start = ISOweek2date(str_glue("{year}-{week}-1"))) %>%
  arrange(week_start)

sales_weekly <- sales_weekly_by_customer %>%
  group_by(customer_acquisition_channel, year, week, week_start) %>% 
  summarise(across(where(is.double), sum), .groups = "drop")

sales_yearly <- sales_weekly_by_customer %>%
  group_by(year) %>%
  summarise(across(c(net_revenue, gross_revenue, boxes), sum))

Modelo ARIMA

Definição

um modelo auto-regressivo integrado de médias móveis (AutoregRessive Integrated Moving Average ou ARIMA, na sigla em inglês) é uma generalização de um modelo auto-regressivo de médias móveis (ARMA).

Um processo ARIMA(1, 1, 2) é formalizado:

  • p: the number of autoregressive terms Y_{t-1}
  • d: the number of nonseasonal differences ϕ_{1} (Y_{t-1} - Y_{t-2})
  • q: the number of moving-average terms θ_{1}e_{t-1} - θ_{2}e_{t-2}

\hat{Y}_t = Y_{t-1} + ϕ_{1} (Y_{t-1} - Y_{t-2}) - θ_{1}e_{t-1} - θ_{2}e_{t-2}

model_arima <- arima_reg() %>%
  set_engine("auto_arima") %>%
  fit(revenue ~ week_start, training(splits))

Prophet

O Prophet foi desenvolvido pelo Facebook que implementa um procedimento para prever dados de séries temporais com base em um modelo aditivo em que as tendências não lineares se ajustam à sazonalidade anual, semanal e diária, além dos efeitos de feriados.

Pertence à família de modelos aditivos, onde a série temporal é resultante da soma de seus componentes:

Y(t) = g(t) + h(t) + s(t)+ e(t)

  • g(t): trend factor
  • h(t): holyday component (world cup, black friday, elections, etc.)
  • s(t): seasonality component
model_prophet <- prophet_reg(
    seasonality_yearly = TRUE
  ) %>%
  set_engine("prophet") %>%
  fit(revenue ~ week_start, training(splits))

GLM

Os Modelos Lineares Generalizados (GLMs) nasceram do desejo de trazer, sob um mesmo guarda-chuva, uma ampla variedade de modelos de regressão que abrangem o espectro de Modelos de Regressão Linear Clássica até modelos para dados baseados em contagens, como Logit, Probit e Poisson, modelos para análise de sobrevivência.

A vantagem do uso do GLM é proporcionar o uso de uma técnica de treino comum para uma variedade de modelos. É possível, portanto, expressar a relação entre X e Y por meio de um modelo linear aditivo.

Aqui, representamos o nosso modelo da seguinte forma:

net\;revenue = week\;day + week\;month + week\;year + numeric\;trend\;feature

model_glmnet <- linear_reg(penalty = 0.01) %>%
  set_engine("glmnet") %>%
  fit(
    revenue ~ lubridate::wday(week_start, label = TRUE) + 
      lubridate::month(week_start, label = TRUE) + 
      lubridate::year(week_start) + as.numeric(week_start),
    training(splits)
  )

Comparando os modelos (forecasts KPI)

  • Mean Absolute Error (MAE): é meramente a média dos erros (diferença entre valores previstos e valores reais), considera-se valores absolutos para não anulação de valores residuais negativos e positivos.
  • Mean Absolute Percentage Error (MAPE): trata-se de um indicador de acerto de estimativa de venda que varia entre 0% e 100%.
  • Symmetric Mean Absolute Percentage Error (sMAPE):De interpretação semelhante ao anterior, porém controlando efeitos de valores extremos.
  • Mean Absolute Scaled Error (MASE): é uma métrica de erro sem escala que fornece cada erro (MAE) como uma proporção em comparação com o erro médio de uma linha de base (naive random walk model). Quanto menor o valor, melhor.
  • Root Mean Squared Error (RMSE): é a raiz quadrática média” dos erros entre valores observados (reais) e predições (hipóteses). uma métrica que nos informa a distância média entre os valores previstos do modelo e os valores reais no conjunto de dados. Quanto menor, melhor.
  • R2: determina a proporção de variância na variável dependente que pode ser explicada pela variável independente. Em outras palavras, r-quadrado mostra quão bem os dados se ajustam ao modelo de regressão (a qualidade do ajuste).

#### Modelo ARIMA

##### Definição

> um modelo auto-regressivo integrado de médias móveis (**A**utoreg**R**essive **I**ntegrated **M**oving **A**verage ou ARIMA, na sigla em inglês) é uma generalização de um modelo auto-regressivo de médias móveis (ARMA).

Um processo $ARIMA(1, 1, 2)$ é formalizado:

- $p$: the number of autoregressive terms $Y_{t-1}$
- $d$: the number of nonseasonal differences $ϕ_{1} (Y_{t-1} - Y_{t-2})$
- $q$: the number of moving-average terms $θ_{1}e_{t-1} - θ_{2}e_{t-2}$

$$\hat{Y}_t   =   Y_{t-1}  +  ϕ_{1} (Y_{t-1} - Y_{t-2}) - θ_{1}e_{t-1} - θ_{2}e_{t-2}$$

```{r}
#| code-fold: false

  set_engine("auto_arima") %>%
```

#### Prophet

> O *Prophet* foi desenvolvido pelo Facebook que implementa um procedimento para prever dados de séries temporais com base em um modelo aditivo em que as tendências não lineares se ajustam à sazonalidade anual, semanal e diária, além dos efeitos de feriados.

Pertence à família de modelos aditivos, onde a série temporal é resultante da soma de seus componentes:

$$Y(t) = g(t) + h(t) + s(t)+ e(t)$$

- $g(t)$: trend factor
- $h(t)$: holyday component (world cup, black friday, elections, etc.)
- $s(t)$: seasonality component

#### GLM

> Os Modelos Lineares Generalizados (GLMs) nasceram do desejo de trazer, sob um mesmo guarda-chuva, uma ampla variedade de modelos de regressão que abrangem o espectro de Modelos de Regressão Linear Clássica até modelos para dados baseados em contagens, como Logit, Probit e Poisson, modelos para análise de sobrevivência.

A vantagem do uso do GLM é proporcionar o uso de uma técnica de treino comum para uma variedade de modelos. É possível, portanto, expressar a relação entre $X$ e $Y$ por meio de um modelo linear aditivo.

Aqui, representamos o nosso modelo da seguinte forma:

$$net\;revenue = week\;day + week\;month + week\;year + numeric\;trend\;feature$$

### Comparando os modelos (forecasts KPI)


- **Mean Absolute Error (MAE)**: é meramente a média dos erros (diferença entre valores previstos e valores reais), considera-se valores absolutos para não anulação de valores residuais negativos e positivos.
- **Mean Absolute Percentage Error (MAPE)**: trata-se de um indicador de acerto de estimativa de venda que varia entre 0% e 100%.
- **Symmetric Mean Absolute Percentage Error  (sMAPE)**:De interpretação semelhante ao anterior, porém controlando efeitos de valores extremos.
- **Mean Absolute Scaled Error (MASE)**: é uma métrica de erro sem escala que fornece cada erro (MAE) como uma proporção em comparação com o erro médio de uma linha de base (*naive random walk model*). Quanto menor o valor, melhor.
- **Root Mean Squared Error (RMSE)**: é a raiz quadrática média" dos erros entre valores observados (reais) e predições (hipóteses). uma métrica que nos informa a distância média entre os valores previstos do modelo e os valores reais no conjunto de dados. Quanto menor, melhor.
- **R<sup>2</sup>**: determina a proporção de variância na variável dependente que pode ser explicada pela variável independente. Em outras palavras, r-quadrado mostra quão bem os dados se ajustam ao modelo de regressão (a qualidade do ajuste).

